### Description

While basic user creation is supported via `seerr_user` and the general `permissions` bitmask, Overseerr also has an endpoint for setting granular, user-specific permissions natively in the UI (e.g., specifically toggling "Auto-Approve Movies", "Auto-Approve Series", or "Request 4K").

### Proposed Solution

Create a `seerr_user_permissions` resource that maps to the `/user/{userId}/settings/permissions` API endpoints.

### High-Level Guidance

- This is distinct from the bitmask `permissions` on the `seerr_user` resource.
- A dedicated resource allows users to module-ize user creation seamlessly. For example, creating a standard user vs a VIP user who gets an attached `seerr_user_permissions` resource granting auto-approval for 4K content.
- The schema should cover boolean toggles for the various auto-approve and management scenarios that Overseerr supports natively (e.g. `auto_approve_movies`, `auto_approve_series`).
