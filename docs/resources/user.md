# seerr_user (Resource)

Manage Seerr users and their notification settings.

## Example Usage

### Import User from Plex

```terraform
# Look up the Plex ID from the Plex users settings endpoint, or use a known Plex ID.
resource "seerr_user" "plex_imported" {
  username = "plex_user"
  email    = "plex@example.com"
  plex_id  = "12345678" # This triggers parsing from Plex

  permissions = 0
}
```

### Basic Local User

```terraform
resource "seerr_user" "basic" {
  username    = "john_doe"
  email       = "john@example.com"
  permissions = 0
}
```

### Admin User with Discord Notifications

```terraform
resource "seerr_user" "admin" {
  username    = "admin_user"
  email       = "admin@example.com"
  permissions = 2 # 2 is MANAGE_SETTINGS (refer to Overseerr docs for bitmask)

  notification_settings = {
    discord_enabled = true
    discord_id      = "123456789012345678"

    notification_types = {
      discord = 2 # 2 is for "REQUEST" notifications
    }
  }
}
```

### User with Telegram and Email Notifications

```terraform
resource "seerr_user" "power_user" {
  username    = "power_user"
  email       = "power@example.com"
  permissions = 32 # Example: MANAGE_USERS

  notification_settings = {
    email_enabled    = true
    telegram_enabled = true
    telegram_chat_id = "987654321"

    notification_types = {
      email    = 1 # 1 is for "GENERIC"
      telegram = 2 # 2 is for "REQUEST"
    }
  }
}
```

### User with Pushover and Custom Sound

```terraform
resource "seerr_user" "pushover_user" {
  username = "pushover_user"
  email    = "pushover@example.com"

  notification_settings = {
    pushover_application_token = "uQiRzpo4DXvYvpt166Y24Y6S1zK6"
    pushover_user_key          = "gzG9no6495478Z99859A83069B6"
    pushover_sound             = "magic"

    notification_types = {
      pushover = 2
    }
  }
}
```

## Schema

### Required

- `email` (String) User's email address. Field is ForceNew because Overseerr API doesn't support updating it.
- `username` (String) User's display name. Can be imported from Plex if `plex_id` is provided.

- `notification_settings` (Block, Optional) User-specific notification settings. (see [below for nested schema](#nestedblock--notification_settings))
- `permissions` (Number) Permissions bitmask.
