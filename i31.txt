## Summary
Add a reproducible integration/acceptance environment so the provider can be validated on every PR without relying on a manually managed external Seerr instance.

## Why
The current OpenTofu test job skips when `SEERR_URL` is unset. That means the most realistic test coverage is optional in CI.

## Proposed changes
- Add a disposable Seerr test environment for CI.
- Prefer `docker-compose` or testcontainers for Seerr plus the minimum mocked/real ARR dependencies needed by the provider.
- Keep the current external-instance path as an override if desired, but make the default CI path self-contained.
- Expand end-to-end coverage for resources that already exist but are not fully exercised in acceptance tests.
- Cover import behavior and no-drift plan behavior for more resources.

## Scope
This issue groups test environment setup and wider acceptance coverage because the second depends on the first.
